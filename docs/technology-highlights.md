## 1. 六层架构

1. **大模型能力层** — 理解、推理、多轮生成、人格一致性；可输出驱动友好的结构化信号。  
2. **人格引擎层** — 人设、语气、边界、犟嘴强度、亲密度与长度控制；**把「随机聪明」收敛为「稳定角色」**。  
3. **对话记忆层** — 短长期记忆、情绪与关系记忆（产品化分期）。  
4. **语音合成层** — 多音色 TTS、分句/流式、情绪韵律、缓存与加速。  
5. **口型与动作驱动层** — viseme / BlendShape / Live2D 参数 / 骨骼状态机。  
6. **宿主与分发层** — App / Web / 桌面 / 直播插件 / 企业大屏；**与核心引擎解耦**。

---

## 2. 人格引擎（配置化）

人格引擎的目标：**限制自由度，提升一致性**。示意 JSON（概念级，非接口契约）：

```json
{
  "personality": "犟嘴型",
  "tone": "轻微吐槽、嘴硬但关心",
  "reply_speed": "fast",
  "emotion_style": "活泼",
  "safety_level": "strict",
  "allow_teasing": true,
  "allow_attack": false
}
```

引擎与模型协同：**模型负责生成能力，引擎负责角色与红线**；高风险场景可自动切到「精修」或加强守卫。

---

## 3. 多模式回复系统

| 模式 | 工程取向 | 用户价值 |
|------|------------------|----------|
| **极速** | 真流式、低首包、适合高频闲聊 | 「像秒回的朋友」 |
| **精修人格** | 先整合再下发，人格守卫更强 | 「像稳定的 IP 角色」 |
| **推理** | 更深上下文与链式思考 | 「像认真的军师」 |

三种模式可与多人格、多模型算力档**正交组合**，支撑会员与 B 端分层。

---

## 4. 语音与口型：闭环体验

- **多音色目录**：与人格 SKU 绑定，支持试听与偏好记忆。  
- **分句合成**：平衡首句延迟与听感自然度。  
- **口型时间轴**：基于 viseme / 音素对齐，把时间戳误差压到体感不可感（目标）。  
- **软降级**：网络或引擎波动时，文本与兜底播报仍可**不断会话**。

---

## 5. 数字人驱动协议

将一次回复抽象为可被宿主消费的 bundle：

- 自然语言正文 
- `emotion` / `expression` / `gesture` / `voice_style`
- 音频 URL 或流式分片 + 口型事件序列  

**2D**：`ParamMouthOpenY`、`ParamMouthForm` 等参数映射 + 平滑。  
**3D**：BlendShape、骨骼动画层、镜头与场景状态机（可接 Unity / Unreal / WebGL）。

---

## 6. 安全与治理

- **内容安全**：多层策略 + 红队回归。  
- **人格安全**：可调犟嘴强度，**严禁**人身攻击与羞辱。  
- **专业边界**：医疗/法律等场景引导至专业人员。  
- **观测**：按人格 / 模式 / 版本打点，驱动持续训练与熔断。

---

## 7. 与传统实现的差异

不是「接一个 LLM + 接一个 TTS」的拼接，而是 **「人格引擎 + 对齐模型 + 语音 + 驱动协议」一体化** 的数字人 OS。
