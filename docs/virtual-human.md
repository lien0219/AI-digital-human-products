# 虚拟人：2D / 3D 与驱动链路 · Hardliner Partner AI

> **产品向说明**：数字人是平台的「面孔」与「身体」，与**人格引擎、语音、大模型**同一套叙事。  
> 不强调任何单一宿主；宿主仅作能力验证。

---

## 1. 体验目标

让用户感到：

1. **在场**：角色持续处于画面中，而不是偶尔弹一段动画。  
2. **在说话**：口型、呼吸、微表情与音频节奏一致。  
3. **在互动**：语气变化能触发动作与表情。  
4. **在关系**：长期记忆与偏好上线后，像「认得你」的伙伴。

---

## 2. 2D 数字人

**适合**：快速上线、强二次元表现力、相对可控成本。

**能力清单**

- 待机 / 微动、眨眼、呼吸  
- 表情切换、头部跟随  
- **口型同步**  
- 身体动作、情绪动作、聊天状态（思考 / 倾听 / 吐槽 / 安慰）

**情绪—动作联想（示例）**

- 吐槽：叉腰、歪头、皱眉、摆手、轻摇头  
- 安慰：微笑、点头、前倾、温柔表情、慢节奏手势

## 3. 3D 数字人

**适合**：虚拟主播、品牌展厅、VR/AR、高端陪伴 App。

**能力清单**

- PBR / 卡通渲染管线  
- 骨骼动画、状态机  
- BlendShape / Morph 表情  
- 口型与情感联动  
- 镜头语言与场景切换  

**远期愿景**

AI 角色 + 可进入的 3D 空间 + 语音 + 记忆 + 社交——**平台即迷你元宇宙引擎的一部分**。

---

## 4. 口型与驱动：从音频到参数

**输入**：TTS 音频 + viseme / 音素时间轴（或引擎估计）。  
**处理**：时间对齐、参数曲线平滑、段落起止重置（避免「张嘴发呆」）。  
**输出**：

- **2D**：`ParamMouthOpenY`、`ParamMouthForm` 等  
- **3D**：嘴部 BlendShape 权重 + 下颌骨骼  

---

## 5. 文本与表演：可读 vs 可演

- **气泡**：可保留轻量表演描写与 Markdown。  
- **播报**：走清洗链路，**不读出**舞台提示符、星号与多余标点。  
- **驱动**：从「清洗后正文 + 情绪标签」映射动作，避免「念台词像在背稿」。
